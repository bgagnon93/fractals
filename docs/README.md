# Fractals — deep-zoom explorer

A GPU-accelerated, effectively-infinite-zoom explorer for the Mandelbrot set,
Julia sets, and the Multibrot family (`zᵈ + c`). Rendering runs in WebGPU
compute shaders; the high-precision math behind deep zoom runs in Rust compiled
to WebAssembly.

It zooms to ~**1e66** before hitting the next precision frontier — far past where
naïve `float` (~1e5) or even emulated `double` (~1e13) give out.

---

## Prerequisites

- **Node.js** 18+ and npm.
- **A WebGPU-capable browser** (recent Chrome, Edge, Firefox, or Safari) with
  hardware acceleration enabled.
- **Rust** (stable) with the `wasm32-unknown-unknown` target, plus
  [`wasm-pack`](https://rustwasm.github.io/wasm-pack/).

Install the Rust pieces:

```bash
# Rust toolchain (if not already installed)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"          # <-- needed in every fresh shell

rustup target add wasm32-unknown-unknown
curl -fsSL https://rustwasm.github.io/wasm-pack/installer/init.sh | sh
```

> **Gotcha:** if `cargo`/`rustc` aren't found, run `source "$HOME/.cargo/env"`.
> Building the WASM crate requires this in any shell that hasn't sourced it.

## Setup & run

```bash
npm install         # JS/TS dependencies
npm run build:wasm  # compile the Rust crate -> src/wasm/ (REQUIRED before dev)
npm run dev         # start the Vite dev server (http://localhost:5173)
```

> **Important:** `src/wasm/` is generated and git-ignored, so a fresh checkout
> has no WASM module yet. **Run `npm run build:wasm` before `npm run dev` or
> `npm run build`**, or the app will fail to import `./wasm/perturbation.js`.
> Re-run it whenever you change anything under `perturbation/`.

| Script              | What it does                                                        |
| ------------------- | ------------------------------------------------------------------- |
| `npm run dev`       | Vite dev server with hot reload.                                    |
| `npm run build`     | Type-check (`tsc`) and bundle to `dist/`. Does **not** build WASM.  |

## How it works (in brief)

**Precision tiers**, selected automatically by zoom depth:

| Zoom         | Tier                                   |
| ------------ | -------------------------------------- |
| < ~1e4       | `f32` per pixel                        |
| < ~1e12      | emulated double-float (`df64`) per pixel |
| ≥ ~1e12      | **perturbation** + arbitrary-precision reference |

**Perturbation** is what unlocks deep zoom: instead of iterating every pixel from
scratch, the CPU (`perturbation`) computes one high-precision *reference orbit*, and
the GPU iterates only the small *delta* of each pixel from it — in plain `f32`,
since the deltas stay small. The absolute center never reaches the GPU; only the
small per-pixel offset does. Extras that make it robust and fast:

- **Rescaled deltas** keep the perturbation values inside `f32`'s exponent range
  past zoom ~1e35 (out to ~1e66).
- **Rebasing** re-anchors a pixel against the reference's closest approach to the
  origin when its delta outgrows its value (Zhuoran's method, generalized so it
  works for Julia/Multibrot too).
- **Reference caching** reuses the (expensive) reference orbit across small
  pans/zooms; the GPU gets a small offset to compensate. The HUD shows `↻` on a
  recompute frame and `·cached` on a reuse frame.

**Fractals are pluggable.** A fractal is one WGSL module (`src/shaders/fractals/`)
defining `fractal_step` / `fractal_step_df` / `fractal_pstep`, plus a matching
`fractal_step` in `perturbation/src/lib.rs`. The drivers and all precision/
perturbation machinery are formula-agnostic. Mandelbrot and Julia share the
`z²+c` module (they differ only in whether the screen maps to `c` or to the seed
`z₀`); Multibrot uses a power-parameterized module.

### Why the `perturbation` crate is required

It is the deep-zoom engine, not an optional add-on:

1. It **stores the view center at arbitrary precision** and accumulates pan/zoom
   into it — impossible with JS `number` (f64), which is why panning would
   otherwise freeze around zoom 1e16.
2. It **computes the high-precision reference orbit** (and rebase index) that the
   perturbation shader iterates against.

Remove it and the app is capped at the double-float tier (~1e13). It serves the
reference orbits for every fractal (Mandelbrot, Julia, Multibrot), not just one.

## Adding a fractal

For an analytic formula `f(z, c)`:

1. Create `src/shaders/fractals/<name>.wgsl` with `fractal_step` (f32),
   `fractal_step_df` (double-float), and `fractal_pstep` (the perturbed
   recurrence — derive `f(Z+ε, …) − f(Z, …)` and write it in the rescaled
   delta space; see `multibrot.wgsl` for the pattern).
2. Add the bigfloat counterpart `fractal_step` in `perturbation/src/lib.rs`.
3. Register it in `src/fractals.ts` and add a scene preset.

Non-analytic formulas (Burning Ship, Tricorn) additionally need care in the
perturbation step (absolute-value / conjugate terms and their glitch handling).
